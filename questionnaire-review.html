Problem & Solution

Current state vs. proposed consolidation

Current State — Confusing
Deal Desk: Legal InfoSec / GRC Request
  • AI Questionnaire, Privacy Questionnaire, and InfoSec Questionnaire live inside Deal Desk: Legal — a contract-focused case type
  • Customer Security Questionnaire (CSQ) lives inside InfoSec / GRC — a separate case type
  • Sales reps don't know which case type to open for a questionnaire request
  • No consistent field set, no automatic routing
→
Proposed State — Centralized
Customer Questionnaire AI Q (from Deal Desk)
  • One case type handles all 4 questionnaire subtypes
  • Sales reps pick "Customer Questionnaire" and select the subtype — routing happens automatically
  • Questionnaire values removed from Deal Desk: Legal and InfoSec / GRC picklists so they're no longer selectable there
  • Existing cases on old record types are untouched

Migration Steps

How questionnaire types get removed from existing case types and added to the new one

1 Restrict picklist on Deal Desk: Legal

Remove questionnaire values from the Agreement_Type__c picklist for the Deal Desk: Legal record type. New cases will no longer be able to select these values. Existing cases are unaffected.

Remove:
AI Questionnaire Privacy Questionnaire InfoSec Questionnaire CSQ
2 Restrict picklist on InfoSec / GRC Request

Same change for InfoSec / GRC Request. The CSQ type moves to the new Customer Questionnaire case type exclusively.

Remove:
Customer Security Questionnaire (CSQ) AI Questionnaire Privacy Questionnaire InfoSec Questionnaire
3 Add values to new Customer Questionnaire RT

The same picklist values become available exclusively on the new Customer Questionnaire record type, keeping picklist values in sync with naming.

Add:
Customer Security Questionnaire (CSQ) AI Questionnaire Privacy / Data Questionnaire InfoSec Questionnaire
4 Existing cases stay intact

The 239 questionnaire cases created on old record types in the last year — and all historical cases — are completely unaffected. Picklist restrictions only apply to new case creation going forward.

239 questionnaire cases on old RTs → untouched
New questionnaire cases → Customer Questionnaire RT only
Old RTs → no longer show Q types in picklist
